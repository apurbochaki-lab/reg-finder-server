## 🚀 Key Features

* **Student Registration API:** A robust POST endpoint to validate and store student details (name and registration number) securely in the database.
* **Proximity Search Logic:** A specialized GET endpoint that uses smart database queries to fetch the requested student's data along with their adjacent neighbors (Previous and Next seats).
* **Data Validation & Error Handling:** Strict server-side validation to ensure no empty or duplicate entries, returning meaningful error messages and HTTP status codes.
* **CORS Enabled:** Configured with Cross-Origin Resource Sharing (CORS) to safely communicate with the frontend application.
* **Environment Variable Protection:** Uses `.env` configuration to secure sensitive data such as database connection strings, ports, and allowed origins.
* **Scalable Architecture:** Designed with clean routing and controller structures, making it easy to scale for more student portal features.
